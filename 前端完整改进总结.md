### 改进3：完善的错误处理系统

#### 改进前
- 简单的纯文本错误消息
- 没有错误类型分类
- 缺少解决建议
- 视觉效果单一

```
Error: Invalid URL format. Please provide a valid http/https URL.
```

#### 改进后
- 6种错误类型自动识别
- 友好的错误卡片UI
- 10+错误代码定义
- 针对性解决建议
- 分层错误展示

#### 错误类型

| 类型 | 图标 | 标题 | 触发场景 |
|------|------|------|---------|
| URL错误 | 🌐 | URL输入错误 | URL格式无效 |
| PDF错误 | 📄 | PDF处理错误 | PDF损坏或无文本 |
| 模型错误 | 🤖 | 模型调用错误 | API密钥错误 |
| 网络错误 | 📡 | 网络连接错误 | 连接超时 |
| 通用错误 | ⚠️ | 处理错误 | 其他错误 |
| 图谱错误 | 🔧 | 图谱生成失败 | 可视化失败 |

#### 错误卡片设计

```
┌─────────────────────────────────────┐
│         🌐 (3em大图标)              │
│                                     │
│        URL输入错误 (标题)            │
│                                     │
│  ╔════════════════════════════════╗ │
│  ║ Error: [url_invalid]          ║ │ ← 错误代码
│  ║ URL格式无效。请提供有效的...   ║ │ ← 详细描述
│  ╚════════════════════════════════╝ │
│                                     │
│  ─────────────────────────────────  │
│  💡 建议：                          │ ← 解决建议
│  检查URL格式是否正确，              │
│  确保使用http://或https://开头      │
└─────────────────────────────────────┘
```

#### 错误代码列表

**输入验证错误：**
- `[input_missing]` - 未提供输入内容
- `[url_invalid]` - URL格式无效

**PDF处理错误：**
- `[pdf_*]` - PDF处理相关错误
- `[pdf_empty]` - PDF无文本内容

**URL处理错误：**
- `[url_*]` - URL获取相关错误
- `[content_empty]` - 内容为空

**处理错误：**
- `[processing_failed]` - 处理过程失败
  - 附加提示：API密钥、超时、配额、权限、网络

#### 智能错误识别

```python
# 系统会自动分析错误消息，识别关键词：
if "URL" in error_message:
    → URL错误类型
elif "PDF" in error_message:
    → PDF错误类型
elif "api" in error_message or "model" in error_message:
    → 模型错误类型
elif "network" in error_message or "connection" in error_message:
    → 网络错误类型
else:
    → 通用错误类型
```
