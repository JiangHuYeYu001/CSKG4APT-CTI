# CTINexus 前端错误处理改进说明

## 📋 改进概述

为了提升用户体验，我为CTINexus的前端UI添加了完善的错误信息展示系统。现在所有错误都会以友好、专业的方式展示，帮助用户快速定位和解决问题。

---

## 🔍 错误识别逻辑

### 1. Pipeline错误（在run_pipeline函数中）

#### 输入验证错误
```python
# 错误代码：input_missing
if not text and not source_url and not pdf_file:
    return "Error: [input_missing] 请输入CTI文本、提供报告URL或上传PDF文件。"

# 错误代码：url_invalid
if source_url and not is_valid_source_url(source_url):
    return "Error: [url_invalid] URL格式无效。请提供有效的http://或https://开头的URL地址。"
```

#### PDF处理错误
```python
# 错误代码：pdf_{error_code}
if pdf_result.get("status") != "success":
    return f"Error: [pdf_{error_code}] {error_message}"

# 错误代码：pdf_empty
if not text:
    return "Error: [pdf_empty] PDF文件中未能提取到文本内容。"
```

#### URL处理错误
```python
# 错误代码：url_{error_code}
if url_source_result.get("status") != "success":
    return f"Error: [url_{error_code}] {error_message}"

# 错误代码：content_empty
if not text:
    return "Error: [content_empty] 未找到可用的报告内容。"
```

#### 通用异常错误
```python
# 错误代码：processing_failed
# 附加提示：
# - API密钥问题
# - 超时问题
# - 配额限制
# - 权限问题
# - 网络问题
```

### 2. 前端错误分类（在process_and_visualize函数中）

```python
# 自动识别错误类型
if "URL" in error_message or "url" in error_message:
    error_type = "url_error"
    error_icon = "🌐"
    error_title = "URL输入错误"

elif "PDF" in error_message or "pdf" in error_message:
    error_type = "pdf_error"
    error_icon = "📄"
    error_title = "PDF处理错误"

elif "model" in error_message.lower() or "api" in error_message.lower():
    error_type = "model_error"
    error_icon = "🤖"
    error_title = "模型调用错误"

elif "network" in error_message.lower() or "connection" in error_message.lower():
    error_type = "network_error"
    error_icon = "📡"
    error_title = "网络连接错误"

else:
    error_type = "unknown"
    error_icon = "⚠️"
    error_title = "处理错误"
```

## 💡 针对性建议

系统会根据错误类型自动提供相应的解决建议：

| 错误类型 | 自动建议 |
|---------|---------|
| **url_error** | 检查URL格式是否正确，确保使用http://或https://开头 |
| **pdf_error** | 检查PDF文件是否损坏，确保文件可以正常打开 |
| **model_error** | 检查API密钥是否正确，确认模型名称是否有效 |
| **network_error** | 检查网络连接，确认可以访问外部服务 |
| **unknown** | 请检查输入内容和配置，或联系技术支持 |

## 📊 错误处理流程

### 流程图

```
用户点击"运行"
    ↓
输入验证
    ├─ 有误 → 返回错误码: input_missing / url_invalid
    └─ 通过 ↓
处理PDF/URL
    ├─ 失败 → 返回错误码: pdf_{code} / url_{code} / content_empty
    └─ 成功 ↓
CSKG4APT提取
    ├─ 失败 → 返回错误码: processing_failed + 提示
    └─ 成功 ↓
生成结果和图谱
    ├─ 图谱失败 → 显示橙色警告（数据可用，图谱不可用）
    └─ 完全成功 → 显示正常结果
```

### 错误返回格式

所有pipeline错误都使用统一格式：
```
Error: [error_code] 错误描述信息 [可选提示]
```

示例：
```python
"Error: [url_invalid] URL格式无效。请提供有效的http://或https://开头的URL地址。"
"Error: [pdf_empty] PDF文件中未能提取到文本内容。请确认PDF包含可提取的文本而非纯扫描图片。"
"Error: [processing_failed] API key not found [提示: 请检查API密钥配置]"
```
