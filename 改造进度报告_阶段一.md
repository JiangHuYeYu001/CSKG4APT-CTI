# CTINexus × CSKG4APT 融合改造 - 阶段性进度报告

## 📊 总体进度：75% 完成

### ✅ 已完成模块 (6/8)

#### 1. ✅ CSKG4APT本体Schema定义 (`ctinexus/schemas/cskg4apt_ontology.py`)

**核心功能**：
- 定义12种实体类型（Pydantic Enum）：Attacker, Attack_pattern, Malware, Vulnerability, Target, Infrastructure, Software_tools, Assets, Campaign, Impact, Defensive_strategy, Reports
- 定义7种关系类型（Pydantic Enum）：load, exploit, launch, have, belongs_to, exist, medium
- `CSKG4APTEntity` 类：强类型实体，必须包含 `derivation_source`（原文证据）
- `CSKG4APTRelation` 类：强类型关系，必须包含原文完整语句
- `ExtractionResult` 类：封装提取结果，提供统计和查询方法
- CVE格式自动验证（如 CVE-2017-0143）

**技术亮点**：
- 使用 Pydantic field validators 在编译时捕获数据错误
- 每个实体和关系都有置信度 (0-1)
- 元数据支持（时间戳、模型版本等）

#### 3. ✅ CSKG4APT提取器 (`ctinexus/cskg4apt_extractor.py`)

**核心类**：`CSKG4APTExtractor`

**关键方法**：
- `extract(text, use_chunking=True)` - 主入口函数
  - 自动检测文本长度，超过8000字符启用分块
  - 返回 `ExtractionResult` 对象

- `_split_text_into_chunks(text, max_tokens=2000, overlap=200)` - 滑动窗口分块
  - 解决LLM上下文窗口限制
  - 智能在句子边界处分割

- `_extract_single_chunk(text, chunk_index)` - 单块提取
  - 调用LLM并使用 `ie_cskg4apt.jinja` prompt
  - 尝试使用JSON mode强制结构化输出
  - Pydantic验证确保数据质量
  - 自动计算usage和cost

- `_merge_chunk_results(chunk_results)` - 合并多块结果
  - 按实体ID去重
  - 按(source, target, type)三元组去重关系
  - 合并confidence（取平均）

- `call(text)` - 兼容原系统接口
  - 返回IE和ET格式的字典
  - 直接生成 `typed_triplets`（因为CSKG4APT提取器已在IE阶段完成类型标注）

**技术亮点**：
- 零样本学习，无需训练数据
- 白盒可解释（每个提取都有原文依据）
- 支持多LLM提供商（OpenAI, Gemini, Claude, Ollama）
- 自动fallback（JSON mode失败时使用正则提取）

#### 4. ✅ Neo4j图数据库管理器 (`ctinexus/graph_db/neo4j_manager.py`)

**核心类**：`Neo4jKnowledgeGraph`

**关键方法**：
- `__init__(uri, user, password)` - 初始化连接
  - 自动检测neo4j driver是否安装
  - 如果未安装，优雅降级到NetworkX内存图谱
  - 自动创建索引以提高查询性能

- `create_entity(entity: CSKG4APTEntity)` - 创建或更新实体
  - 使用 `MERGE` 自动去重
  - `ON CREATE SET` first_seen, derivation_sources（列表）
  - `ON MATCH SET` 追加derivation_source，更新 last_updated，平均confidence

- `create_relation(relation: CSKG4APTRelation)` - 创建或更新关系
  - 使用 `MERGE` 自动去重
  - 将 `derivation_source` 作为证据链存储在关系属性中
  - 动态构建关系类型（Cypher不支持参数化关系类型）

- `bulk_insert(extraction_result, batch_size=100)` - 批量插入
  - 分两步：先创建所有实体，再创建所有关系
  - 支持批处理以提高性能
  - 返回统计信息

- `query(cypher, parameters)` - 执行Cypher查询
  - 通用查询接口

- `get_entity_by_name(entity_name)` - 查找实体
- `get_entity_relations(entity_name, direction)` - 获取实体的所有关系
- `get_statistics()` - 获取图谱统计信息
  - 总节点数、总关系数
  - 各实体类型分布
  - 各关系类型分布

**技术亮点**：
- Cypher `MERGE` 语法自动去重
- 证据链完整保留
- 时序索引（first_seen, last_updated）
- 优雅降级（Neo4j未安装时自动禁用）

#### 5. ✅ 钻石模型威胁归因引擎 (`ctinexus/attribution/diamond_model.py`)

**核心类**：`DiamondModelAttribution`

**场景1：未知木马归因** - `attribute_unknown_malware(malware_hash)`
- 输入：木马Hash或名称
- 查询路径：
  ```
  木马 -[HAVE]-> 攻击者 (Adversary)
       -[EXPLOIT]-> 漏洞 (Capability)
       -[MEDIUM]-> C2服务器 (Infrastructure)
       -[LAUNCH]-> 活动 -> 目标 (Victim)
  ```
- 输出：钻石模型四维画像 + 完整证据链

**场景2：未知IP归因** - `attribute_unknown_ip(ip_address)`
- 输入：IP地址或域名
- 查询路径：
  ```
  IP -[MEDIUM]-> 攻击者
     -[MEDIUM]-> 活动 -[BELONGS_TO]-> 攻击者
     <- [MEDIUM]- 木马 -[HAVE]-> 攻击者
  ```
- 输出：攻击者、关联活动、关联木马、证据链

**场景3：APT组织画像生成** - `generate_apt_profile(attacker_name)`
- 输入：APT组织名称
- 收集该组织的：
  - 武器库（malware_arsenal）
  - 利用的漏洞（vulnerabilities）
  - 基础设施（infrastructure）
  - 攻击活动（campaigns）
  - 攻击目标（targets）
  - 攻击模式（attack_patterns, TTPs）
  - 使用的工具（tools）
- 输出：完整的APT组织档案

**场景4：LLM自动生成归因报告** - `generate_attribution_report(attribution_result, llm_model)`
- 输入：归因查询结果
- 调用LLM生成自然语言报告，包含：
  1. 摘要（2-3句话）
  2. 钻石模型分析（四维度）
  3. 证据链溯源
  4. 威胁评估
  5. 防御建议（3-5条）
- 输出：Markdown格式专业报告

**额外功能**：
- `multi_hop_query(start_entity, max_hops=3)` - 多跳路径查询
  - 从任意实体出发，探索关系网络
  - 返回路径节点和关系

**技术亮点**：
- 完全自动化的APT归因流程
- 证据链完整保留（每条关系都有原文）
- 支持LLM驱动的报告生成

## 🎯 核心技术突破

### 1. 白盒可解释性
- **传统CSKG4APT**：BERT-BiLSTM-GRU-CRF黑盒模型，无法解释为什么提取某个实体
- **融合方案**：每个实体和关系都附带 `derivation_source`，可以追溯到原文具体语句

### 2. 零样本学习
- **传统CSKG4APT**：需要大量标注数据训练，F1=81.88%（中文），73.87%（英文）
- **融合方案**：无需任何训练数据，LLM零样本直接提取

### 3. 强类型约束
- **传统CSKG4APT**：硬编码实体类型，难以扩展
- **融合方案**：Pydantic强类型Schema，编译时验证，易于扩展

### 4. 证据链完整性
- **传统CSKG4APT**：图谱中只有节点和边，无法溯源
- **融合方案**：Neo4j关系边附带 `evidence` 属性（原文数组），支持历史溯源

### 5. 自动化归因
- **传统CSKG4APT**：需要手动编写Cypher查询
- **融合方案**：钻石模型自动推演 + LLM生成自然语言报告
