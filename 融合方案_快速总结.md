# 融合方案快速总结

## 核心概念

### CSKG4APT (论文)
- **本体**: 12类实体 + 7类关系 (针对APT)
- **算法**: BERT-BiLSTM-GRU-CRF (需要训练)
- **应用**: APT威胁归因、属性分析

### CTINexus (项目)
- **框架**: LLM驱动的4阶段pipeline (IE→ET→EA→LP)
- **优势**: 零样本、多provider、Web UI
- **限制**: 缺少APT专用本体、没有威胁归因

## 关键设计决策

### ❓ 为什么用LLM而不是神经网络?

| 维度 | LLM | 神经网络 (BERT-BiLSTM-CRF) |
|------|-----|--------------------------|
| 学习数据需求 | 0张 | 10000+张 |
| 模型训练时间 | 0 | 数小时/天 |
| 模型更新难度 | 修改Prompt | 重新训练 |
| 多语言支持 | 原生 | 需要多语言模型 |
| **成本** | 按Token计费 | 前期成本 |
| **维护** | 简单 | 复杂 |

**结论**: LLM更适合工程落地 (快速迭代，低维护成本)

### ❓ 为什么必须有Neo4j?

**不是必须的，但是**:
- 支持复杂的图查询 (Cypher)
- 支持知识库持久化 (不用每次重新提取)
- 支持相似度计算 (识别同一APT组织)
- 支持多报告融合 (知识积累)

**可选**: 如果只要立刻见到成果，可以先不要Neo4j，用内存图+文件保存

## 使用示例

### 最简单的用法

```python
from ctinexus import CSKG4APTExtractor

extractor = CSKG4APTExtractor(config)

# 输入CTI文本
text = "APT28 used Zebrocy to exploit CVE-2017-0143..."

# 获得结构化知识图
kg = extractor.call(text)

# 输出
print(kg.entities)      # 12类实体
print(kg.relations)     # 7类关系
print(kg.dict())        # JSON格式
```

### 威胁卡片生成

```python
from ctinexus import APTAttributor

analyzer = APTAttributor()

# 生成APT威胁卡片
threat_cards = analyzer.generate_threat_card(kg)

# 获得Diamond Model
diamonds = analyzer.diamond_model_analysis(kg)

# 输出
print(threat_cards[0].attacker_names)      # ['APT28', 'Fancy Bear']
print(threat_cards[0].malwares)            # ['Zebrocy', ...]
print(threat_cards[0].exploited_cves)      # ['CVE-2017-0143']
```

### Neo4j查询

```python
from ctinexus.graph_db import Neo4jHandler

handler = Neo4jHandler('bolt://localhost:7687', 'neo4j', 'password')

# 保存知识图
handler.create_kg_graph(kg)

# 查询APT组织画像
profile = handler.query_apt_profile("APT28")

# 查询攻击链
chains = handler.find_attack_chain("APT28", "Financial")
```
