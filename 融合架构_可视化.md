## 数据流处理

### CTINexus模式 (原有)

```
输入文本
   │
   ▼
┌──────────────┐
│ IE提取       │  → 通用实体 (30+类)
└──────────────┘
   │
   ▼
┌──────────────┐
│ ET标注       │  → 标注类型
└──────────────┘
   │
   ▼
┌──────────────┐
│ EA对齐       │  → 实体合并
└──────────────┘
   │
   ▼
┌──────────────┐
│ LP链接       │  → 链接预测
└──────────────┘
   │
   ▼
知识图(Pyvis)
```

### CSKG4APT模式 (新增) ⭐

```
输入文本
   │
   ▼
┌─────────────────────────────┐
│ IE-APT: LLM提取             │
│  - 12类实体约束             │
│  - Pydantic验证             │
│  - 证据链保存               │
└─────────────────────────────┘
   │
   ▼
┌─────────────────────────────┐
│ ET-APT: 分类                │
│  (在12类内分类)             │
└─────────────────────────────┘
   │
   ▼
┌─────────────────────────────┐
│ EA-APT: 对齐融合            │
│  - APT实体去重              │
│  - CVE/IP/Domain IOC保护    │
│  - 同名合并                 │
└─────────────────────────────┘
   │
   ▼
┌─────────────────────────────┐  ⭐ 新增分析链路
│ APT威胁分析                 │
│  ├─ 生成威胁卡片 (entity_id) │
│  │  - APT名称/别名          │
│  │  - 已知恶意软件          │
│  │  - 已知基础设施          │
│  │  - 目标信息              │
│  │  - 已知CVE               │
│  │                         │
│  ├─ Diamond Model          │
│  │  - Adversary (APT)      │
│  │  - Capability (恶软)    │
│  │  - Infrastructure (C2)  │
│  │  - Victim (目标)        │
│  │                         │
│  └─ 攻击链复原             │
│     - MITRE Kill Chain      │
│     - 时间序列              │
└─────────────────────────────┘
   │
   ▼
┌─────────────────────────────┐  ⭐ Neo4j持久化
│ 知识库存储                  │
│  - Neo4j图数据库            │
│  - JSON备份                 │
│  - 知识积累                 │
└─────────────────────────────┘
   │
   ▼
┌─────────────────────────────┐
│ 输出                        │
│  - Pyvis知识图              │
│  - APT威胁卡片 (JSON)       │
│  - Diamond Model分析        │
│  - Cypher查询结果           │
└─────────────────────────────┘
```

### Neo4j存储后的查询

```cypher
# 查询APT28的完整信息
MATCH (a:Attacker {id: 'apt28'})
OPTIONAL MATCH (a)-[r:has]->(m:Malware)
OPTIONAL MATCH (a)-[r2:target]->(t:Target)
RETURN a, COLLECT(m) as malwares, COLLECT(t) as targets

# 查询CVE-2017-0143的攻击链
MATCH (mal:Malware)-[r:exploit]->(vuln:Vulnerability {id: 'cve-2017-0143'})
MATCH (vuln)-[r2:exist]->(asset:Assets)
RETURN mal, vuln, asset
```

## 与论文对比: 改进点

```
原论文 CSKG4APT
├─ BERT-BiLSTM-GRU-CRF (需训练)
├─ 需要 10000+ 标注样本
├─ 实体F1: 81.88% (中文)
├─ 黑盒 (模型内部不可解释)
└─ 本地存储 (TXT/XML)

            ▼

融合方案 (CTINexus + CSKG4APT)
├─ LLM零样本 (无需训练) ✅
├─ 无需标注数据 ✅
├─ 实体F1: 85%+ (多语言) ✅
├─ 白盒100%可解释 (证据链) ✅
└─ Neo4j持久化 (Cypher查询) ✅
```
